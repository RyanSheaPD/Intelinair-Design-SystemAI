Two web patterns: (1) Spinner — indeterminate loading in widgets, modals, and panels (not for tables). (2) Table bar — linear indeterminate bar only at the top of a data table, above the header; see Data table below. Spinner — indeterminate (loading.css); widgets, modals, inline — not above tables

Table loading uses a linear bar (.loading-linear--table) fixed above the table header — demo in Data table (web).

iOS
Use UIActivityIndicatorView or SwiftUI ProgressView() for modal/widget loading. For table data loads, use a slim top-edge progress (e.g. pinned UIProgressView above the table) rather than centering a spinner on the grid. Keep system tints unless a contained surface explicitly calls for brand color.
Android
Use CircularProgressIndicator for dialogs and blocking loads; pair LinearProgressIndicator with tables at the top when refreshing row data. Prefer default Material motion and sizing; brand tint only where spec calls for it inside a defined surface.

Data table (web)

Aligned with retailer Angular Material MDC tables: Figtree 14px / 400, line-height 17px, text rgba(0,0,0,.87), row border #e6e6e6, cell padding 0 0 0 5px, row transition 225ms cubic-bezier(0.4, 0, 0.2, 1). Link tokens.css, loading.css (for the top bar), then data-table.css.

Loading row data: wrap the table in .ds-data-table-shell. First child: .loading-linear.loading-linear--indeterminate.loading-linear--table (set hidden when idle). Second child: .ds-data-table-shell__scroll around <table class="ds-data-table">. The bar is the only permitted use of the linear loader — use .loading-spinner everywhere else.

Banners and Alerts

Use banners for fixed-width block messages (360px). Use alerts for rectangular message blocks (different size and layout from pills) for inline or toast feedback. Both support a leading status icon and optional dismiss. Variants: success, error, warning, info.

Classes: .banner .banner--success|error|warning|info and .alert .alert--success|error|warning|info. Add .banner__icon / .alert__icon, .banner__text / .alert__text, and optional .banner__dismiss / .alert__dismiss.

Modals

Use as a card (inline panel) or as a popup (overlay). Add .modal__close in the header for an X to dismiss. Dividers are 1px neutral primitive. Use .modal__split + .modal__main / .modal__side for a vertical split layout (lighter top, darker bottom using neutral-100).
