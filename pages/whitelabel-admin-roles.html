Section 1

Executive Overview & Architecture

Current State

The MyFS home page (/retailer/home) is powered by NewDashboardComponent — an Angular component that renders a backend-configured widget grid. The widget catalog is served from ui-conf/widgets, and visibility is governed by the intersection of whitelabel app features and user role-based features.

Key Insight There is no separate admin home template today. Admin vs. regular user differences are entirely data-driven (which widgets the backend returns, plus the growers permission gating on summary stats).

Proposed Architecture

The new home page should be a single, flexible component that adapts based on three axes:

🏷️
Whitelabel Theme

Brand colors, logos, nav, terminology

👤
User Role

Admin, WL Admin, Junior Admin, User, Grower

🚩
Feature Flags

AppFeature keys enable/disable widgets & sections

Brand dashboard shells

Home layout preview using design-system .nav, .header, and SVG assets from ../assets/icons/nav/ and ../assets/icons/timeline/. Pivot Bio header uses ../assets/icons/pb-logo-header.svg.

Section 2

Roles & Feature Flag Framework

👑

Super Admin

ROLE_ADMIN

Full cross-whitelabel access. Sees all widgets, all companies, all settings.

🏢

Company Admin

ROLE_COMPANY_ADMIN

Manages all companies in their hierarchy. User management, contracts, integrations.

🏷️

WhiteLabel Admin

ROLE_WHITE_LABEL_ADMIN

Manages users within their whitelabel. Can assign roles per RoleManagement rules.

📋

Junior Admin

ROLE_JUNIOR_ADMIN

Limited admin with subset of role management capabilities.

Admin-Specific Home Page Behavior (Current) When isCompanyAdmin() or isAdminRole() (WhiteLabelAdmin / JuniorAdmin) and the user has showWizard=true with a default company alias, login redirects to /integration (onboarding wizard) instead of /retailer/home.
Section 3

Epic-Level Requirements — Universal Home Page

🏠
Epic HP-001

Whitelabel-Aware Home Shell

High Priority

Goal: The home page shell (header, sidebar nav, greeting, summary stats) must dynamically apply the correct whitelabel branding (colors, logo, terminology) based on the authenticated user's whitelabel association.

  • HP-001-01: Load brand theme tokens (CSS variables) based on user.whiteLabel.keyName — e.g. nutrivize applies PB Blue palette, holganix_horizons applies AG Green palette. Whitelabel
  • HP-001-02: Header logo, favicon, and document title sourced from WhiteLabel.metadata.templates and ThemePipe asset resolution. Whitelabel
  • HP-001-03: Sidebar navigation items filtered by user's appFeatureKeys (intersection of WL + role features). Feature Flag
  • HP-001-04: Greeting header shows localized "Hello, {firstName}" with company name and date. Feature Flag
  • HP-001-05: Summary stat pills (Growers, Alerts, Fields) conditionally visible — Growers requires growers permission. Role
📊
Epic HP-002

Role-Adaptive Widget Dashboard

High Priority

Goal: The widget grid renders a curated set of widgets based on the combination of whitelabel features and user role. Admins see operational + management widgets; standard users see field-level insights only.

  • HP-002-01: Backend ui-conf/widgets returns widget configs filtered by user's effective features (WL features ∩ role features). Feature Flag
  • HP-002-02: Admin-exclusive widgets: User Management overview, Platform Health, Company Contracts summary, Integration Status. Role New
  • HP-002-03: Shared widgets rendered for all roles: Alerts Overview, Crop Growth, Weather, Fields Outlook, Scouting, Ten Worst Fields. Visibility per WL feature assignment.
  • HP-002-04: Widget layout supports drag-and-drop reordering with persist-to-backend (existing capability). Grid columns adapt: 3-col on desktop, 2-col on tablet, 1-col on mobile.
  • HP-002-05: Partner-specific widgets: hgxVintage (Holganix only), Pivot Bio Product/NFactor widget (Nutrivize only). Whitelabel New
⚡
Epic HP-003

Admin Quick Actions & Shortcuts

Medium Priority New

Goal: Admin-role users get a dedicated "Quick Actions" section on the home page providing fast access to their most common administrative tasks, eliminating extra navigation clicks.

  • HP-003-01: Quick Action cards for: Invite User, Manage Roles, Connect Fields (Integration), View Company Contracts, Manage Feature Flags. Role
  • HP-003-02: Only visible for ADMIN_ROLES (Admin, CompanyAdmin, WhiteLabelAdmin, JuniorAdmin). Specific actions further gated by admin_settings feature flag.
  • HP-003-03: Junior Admins see a subset based on RoleManagement configuration (cannot see role assignment if not authorized). Role
📢
Epic HP-004

Notifications & Communication Hub

Medium Priority

Goal: Surface recent notifications, messages, and communication activity directly on the home page for users with the communications_platform feature flag.

  • HP-004-01: Notification feed widget showing recent alerts, scouting assignments, and system notifications. Gated by communications_platform. Feature Flag
  • HP-004-02: Unread message badge count in sidebar and home widget, pulling from communication-service. Feature Flag
  • HP-004-03: Admin-specific: "Pending User Requests" count for user_request_management feature. Role New
📈
Epic HP-005

Analytics & Campaign Widgets

Medium Priority

Goal: Provide data-rich dashboard widgets for analytics features (Analyze, Carbon MRV, Crop Models) and ad campaign management, each gated by their respective feature flags.

  • HP-005-01: Analyze summary widget visible when analyze feature flag is active. Shows top-line metrics (stress, nutrient deficiency, emergence scores). Feature Flag
  • HP-005-02: Carbon MRV summary widget visible when carbon_mrv is active. Feature Flag
  • HP-005-03: Ad Campaign carousel for ads_banner_widget1 / ads_banner_widget2 (existing). Feature Flag
  • HP-005-04: Market prices & News widgets for partners with markets_widget / latest_news_widget features.
🤖
Epic HP-006

AI Assistant & Intelligent Insights

Medium Priority

Goal: The AI chat assistant (llm_chat / ai_chat feature flags) should have a prominent entry point on the home page, plus contextual insight cards powered by AI analysis.

  • HP-006-01: Floating AI assistant chip on retailer shell (existing — *ngxPermissionsOnly="eFeatureFlag.llmChat"). Feature Flag
  • HP-006-02: "AI Insights" home widget showing auto-generated weekly summary of field health, anomalies, and recommended actions. New
  • HP-006-03: AI sidebar chat entry point in left navigation when ai_chat is enabled.

Pivot Bio Admin Home — Product & Trial Insights

Nutrivize

Goal: Pivot Bio admin users need a home page that combines standard agronomic dashboard capabilities with Pivot Bio-specific product performance and trial management features.

  • PB-001-01: PB Product Performance Widget — New widget displaying N-Factor scores and product trial results from /api/v2/pivotBio/product and /api/v2/pivotBio/nFactor endpoints. Admins see cross-company data; regular users see only their WL-scoped data. New
  • PB-001-02: Trial Management Quick Action — Admin shortcut to view/manage active trials (manage_trials feature flag). Links to /retailer/trials.
  • PB-001-03: PB-Specific Scouting Categories — Scouting widget pre-filtered to Nutrivize-specific scouting categories (biological product observations, N response assessments). Categories seeded via ScoutingCategory:whitelabel:pivotbio.
  • PB-001-04: Nitrogen Management Insights — Summary tile showing N application status across managed fields, leveraging analyze filters (nitrogenPractice). Gated by analyze feature. Feature Flag
  • PB-001-05: Theme Application — PB Blue (#2B3A4D) applied to nav, buttons, stat pill accents, and active states via .theme-pb CSS class.

Widget Resolution Flow

How widgets are determined for the home page dashboard grid:

User authenticates
→
JWT includes app_feature_keys
→
GET /ui-conf/widgets
→
Filter by defaultStyle
→
Render widget grid
Feature Resolution Formula Effective features = (AppFeatures for user's WhiteLabel where isDynamicCalculated=false) ∩ (AppFeatures for user's Roles) + dynamic features (e.g., hide_imagery_data). This intersection ensures a user only sees widgets that both their whitelabel allows and their role grants.
