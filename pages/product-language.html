Growth stage

The current crop development stage as modeled or observed in the field. Growth stage models are crop-specific:

Corn V-stage (VE, V1...VT) and R-stage (R1...R6) system
Soybean V-stage (VE, VC, V1...Vn) and R-stage (R1...R8) system
Wheat Feekes scale (1-11.4) or Zadoks scale (00-99) — user selectable. Includes dormancy and green-up stages.
Use this
Use "growth stage" as the umbrella term. Always display the crop-appropriate scale (V/R for corn and soy, Feekes or Zadoks for wheat).
Don't use this
Don't mix scale systems. Don't show Feekes labels on a corn field or V-stages on wheat.

Section (Template / Recipe)

open_in_new Crop Plan page

A building block that defines a particular management practice — such as a seeding program, herbicide program, or fertilizer plan. Sections are partial, named elements that can be created, stored, and reused. They are the standardized formulas or templates you create before applying them to fields.

Key distinction: A section is not a crop plan. It is a reusable component that becomes part of a crop plan only when applied to a field. Think of it as a recipe — it exists independently until it's used.
Use this
Use "section" as the current product term. "Template" or "recipe" are acceptable informal alternatives when explaining the concept. Three section types: Soil management, Seed, Applications.
Don't use this
Don't call a section a "crop plan" — it's not a plan until it's applied to a field. Don't use "template" in UI if the product label is "section."
Design System Mapping
description Crop Plan page → "Sections" tab (default view) add "+ Create" dropdown: Soil management, Seed, Applications table_view Sections table: Name, Category, Products, Timing, Date created, Date revised, Created by badge Status: Published / Draft

Crop Plan

open_in_new Crop Plan Builder

The overall collection of sections (templates/recipes) that has been applied to a specific field. A crop plan is not a crop plan until it is on the field — it is created by taking planned sections and applying them in bulk to fields. Once applied, the system generates a summary showing the total amount of product needed (seeds, nitrogen, herbicide, etc.) to accomplish the plan.

Relationship: Sections are the building blocks. A crop plan is the completed assembly applied to a field.
Example: A "2026 Corn Standard" crop plan might include 4 sections: Standard Corn (seed), Pre-plant Fertilizer, Herbicide Program, and Fungicide Application.
Use this
Use "crop plan" only when referring to the assembled plan applied to a field. Use "template-based crop plan" when emphasizing reusability. Entry points: Crop Plan nav item and Field Detail View.
Don't use this
Don't call a standalone section a "crop plan." Don't use "spreadsheet plan" once it lives in the product.

Company admin / Permission role

Company admin: company-level administrative role. Permission role: governs what a user can do. Communication group: targeted messaging audience. These are separate concepts.

Three permission levels (managed in Settings > Admin):
U User — standard access to fields, scouting, and reporting
A Admin — user management, settings, and configuration
S Super Admin — full system access, cross-company visibility
Use this
Use "User," "Admin," and "Super Admin" as the three distinct role levels. Roles are managed in the Settings admin section.
Don't use this
Don't use "company admin" loosely when you mean one of these three specific levels. Don't confuse permission roles with communication groups.
Design System Mapping
